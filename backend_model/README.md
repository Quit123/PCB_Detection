# PCB Detection(Model Backend)

## 目录结构

The dataset for this project is organized as follows, designed for the **BJ-PCB** defect detection task:

```plaintext

datasets/
├── BJ-PCB/                            # Pre-split training/validation/testing dataset (for testing)
│   ├── train/                         # Training set
│   │   ├── images/                    # Images
│   │   └── labels/                    # YOLO-format annotations
│   └── val/                           # Validation set
│
├── normal/                            # General dataset (optional, for data augmentation or comparison)
│
├── raw/                               # Raw data files (annotations and original images)
│    ├── BJ-PCB/                       # Raw data files (for testing)
│    │   └── Annotations/              # Original annotation files (organized by defect type)
│    │       ├── Missing_hole/
│    │       ├── Mouse_bite/
│    │       ├── Open_circuit/
│    │       ├── Short/
│    │       ├── Spur/
│    │       └── Spurious_copper/
│    └── next_train/                   # Raw data for the next training iteration
│            ├── images/               # Original images
│            └── labels/               # Converted YOLO-format annotations (generated by scripts)
├── next_train/                        # Next-round training dataset
│    ├── train/                    
│    │   ├── images/                   # Images
│    │   └── labels/                   # YOLO-format annotations
│    └── val/                          # Validation set for the next training round


```

## Notes

* If your GPU driver does not support CUDA 11.8, please visit the [PyTorch](https://pytorch.org/get-started/locally/)
 to select a compatible version.
* The project supports Windows and Linux systems. It is recommended to use Conda for Python environment management.
* If dependency conflicts or installation failures occur during setup, try updating Conda first:

```bash
conda update -n base -c defaults conda
```
